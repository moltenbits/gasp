# gasp-runtime

Minimal runtime library that bridges the generated code to a live `graphql-java` schema. It loads the SDL produced by `gasp-processor`, registers custom scalar types, and wires DataFetchers — all in a few static method calls with no classpath scanning or reflection.

## Dependency

```kotlin
dependencies {
    implementation("com.moltenbits.gasp:gasp-runtime:0.1.0-SNAPSHOT")
}
```

Transitive dependencies: `gasp-annotations`, `graphql-java` (21.3).

## Components

### SchemaLoader

`SchemaLoader` is the single entry point for assembling an executable GraphQL schema from the processor's output.

```java
public static GraphQLSchema load(
    Map<String, DataFetcher<?>> queryFetchers,
    Map<String, DataFetcher<?>> mutationFetchers,
    Map<String, Map<String, DataFetcher<?>>> typeFetchers
)
```

It performs four steps:

1. **Loads the SDL** from `META-INF/gasp/schema.graphqls` on the classpath — the resource file generated by `gasp-processor` during compilation. Throws `IllegalStateException` if the resource is missing (meaning the annotation processor did not run).

2. **Parses** the SDL into a `TypeDefinitionRegistry` using graphql-java's `SchemaParser`.

3. **Builds `RuntimeWiring`:**
   - Registers the `Date` and `DateTime` custom scalars from `ScalarDefinitions`
   - Wires each query fetcher to its field name under the `Query` type
   - Wires each mutation fetcher under the `Mutation` type
   - Wires type-level fetchers (for entity field resolvers and relations) to their respective object types

4. **Returns** an executable `GraphQLSchema` via `SchemaGenerator.makeExecutableSchema()`.

The generated `GaspSchemaRegistry` calls this method — application code typically does not call `SchemaLoader` directly.

### ScalarDefinitions

Defines two custom `GraphQLScalarType` instances used whenever `java.time` types appear in annotated code.

#### `DATE`

Maps `java.time.LocalDate` to the GraphQL `Date` scalar.

- **Serialize** (Java → response): formats as ISO-8601 date string (`2024-01-15`)
- **Parse value** (variable → Java): parses ISO-8601 date string to `LocalDate`
- **Parse literal** (inline query value → Java): expects a `StringValue` AST node, parses to `LocalDate`

#### `DATE_TIME`

Maps multiple Java temporal types to the GraphQL `DateTime` scalar.

- **Serialize** (Java → response): polymorphic — handles `LocalDateTime` (ISO local), `OffsetDateTime` (ISO offset), `ZonedDateTime` (ISO zoned), and `Instant` (`.toString()`)
- **Parse value** (variable → Java): tries `OffsetDateTime` first (preferred, includes timezone), falls back to `LocalDateTime`
- **Parse literal**: same parse strategy as parse value, expects a `StringValue` AST node

All coercing methods throw the appropriate graphql-java exception (`CoercingSerializeException`, `CoercingParseValueException`, `CoercingParseLiteralException`) on invalid input.

## Design principles

- **No reflection, no scanning.** Schema assembly is purely wiring pre-generated artifacts.
- **No framework coupling.** This module depends only on `graphql-java` and `gasp-annotations`. Framework adapters (`gasp-micronaut`, `gasp-spring`) depend on this module and provide the DI and HTTP layers.
- **Stateless.** `SchemaLoader` is a utility class with a single static method. `ScalarDefinitions` exposes two `static final` constants. No instances, no mutable state.
